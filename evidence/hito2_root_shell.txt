Explanation

Since the GitHub Codespaces environment installs Python in non-standard locations, the original script could not find the standard library (stdlib). As a result, within the virtual machine, Python would crash due to encoding errors or missing modules.
Writing the exploit in C and compiling it statically (-static) embedded all the necessary libraries directly into the file itself. This eliminated any dependency on Python, allowing the program to communicate directly with the kernel and execute the attack cleanly.
chmod 4755 was added to the su command and busybox in the packaging script so that, once the exploit tricks the kernel in memory, the operating system has the necessary permissions on the disk to grant you the root prompt.
